Testing Across the Language Barrier with FFI::Platypus

(From 10:00 to 10:25 in COLESVILLE room)

Perl has an excellent testing culture, testing tools and testing libraries. Perl also has an effective Foreign Function Interface (FFI) called FFI::Platypus that allows developers to call into dynamic libraries directly without having to write XS. In this talk I will show you how to use these highly effective ingredients to test across the language barrier in projects that mix-and-match Perl and compiled code. I will use as an example how we used this technique at Fastly to call Varnish Configuration Language (VCL) in unit tests.

~ Graham Ollis

Senior Software Engineer at Fastly

Practical Perl 6 Regular Expressions

(From 10:00 to 10:25 in COLESVILLE room)

This talk covers the basics of Perl 6 regular expressions by going through a number of real world examples. We cover some gotchas for those coming from Perl 5, as well as some little gems that make hard things easy. We will make use of basic syntax, the match variable, interpolation, character classes, predefined rules, and other goodies.

~ Brian Duggan

Likes coding, math, and short fiction. His favorite color is blue.

Multisection: When Bisection Is Not Enough to Debug a Problem

(From 16:30 to 16:55 in FENTON room)

Bisection is a developer's tool for identifying a change set in a codebase where an anomaly such as a test failure first appeared. In complex applications, however, there are situations where you need to identify multiple problematic transitions within the code base. Multiple bisection — multisection, for short — is a tool which has been developed to help debug the Perl 5 core distribution. In this presentation we describe multisection and suggest how you may use it in other projects.

~ James E. Keenan

Author of more than two dozen CPAN modules and contributor to Perl 5 core development team. Speaker at past DC-BPWs.

Introduction to Git for Non-Developers

(From 11:00 to 11:55 in COLESVILLE room)

Git is a revision control system that is used for many Open Source projects. Having a basic understanding of Git is essential being able to join an Open Source project and become a contributor. It's also super useful for many other activities! This talk will explore the basics of Git, assuming no existing background experience. Via analogies to other, familiar technolgies, the basic principles of using Git will be explained in an approachable, understandable fashion. People who attend this talk should come away ready to make an initial contribution to an Open Source project, and will leave with a list of additional resources to explore to learn more.

~ John S.J. Anderson

Adventures in Dunderland: The Python Data Model for Perl Mongers

(From 13:00 to 13:55 in COLESVILLE room)

Python is designed to be easy to learn for beginners, and easy to pick up for experienced programmers. But Python's simplicity means you might not be using all the features the language has to offer. This is especially true if the features don't exist in your favorite programming language. In this talk I'll review the Python data model, which is the key to writing idiomatic Python code. You'll learn how to emulate the behavior of Python's internal types such as sequences and dictionaries using Python's special double-underscore ("dunder") methods.

~ Walt Mankowski

Walt Mankowski is a senior software engineer at Syapse, where he helps healthcare providers bring precision medicine to cancer patients. Walt's a recovering ivory tower computer scientist who enjoys pointing out things that Perl does better than Python to his coworkers. He enjoys Perl, regular expressions, high-performance computing, and Futurama.

Where am I now, Really? A novel use of decompilation‎

(From 11:30 to 11:55 in COLESVILLE room)

Perl was created by a language enthusiast, so it is only fitting to finish the circle and show how computer-to-human translation is like human-to-human translation. More so that I think most people have hitherto appreciated.

Decompiling is often used in conjunction with recovering lost source code, or reverse-engineering code when we do not have access to the source code. It can be optionally used in Data::Printer for code references you have stashed in your data structures.

I will describe a novel use: in tracebacks, debuggers, and similar places where accurate position reporting is desired, even in the presence of string evaluation where the source code was created at runtime.

I'll show how line numbers are vague and ambiguous.

The module B::Deparse has been around a long time for Perl. I have extended that in B::DeparseTree so that it can be used at runtime to show you where you are. I will also demo its use in a real debugger, and promised to include a reference to at least one of the great virtues of programmer espoused by Larry Wall.

~ Rocky Bernstein

I am a long-time CPAN contributor, and open-source developer more generally.

Among the many different kinds of programs I have written are debuggers for 8 or so programming languages, including Perl, bash and GNU make.

I have worked at IBM Research, as a government contractor for NASA/Goddard (in climate studies), the Associated Press, a large financial institution, a university, and way too many startups.

I currently work in the Ethereum cryptocurrency space at ConsenSys.
